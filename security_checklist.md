# 배포 전 보안 체크리스트

새 프로젝트를 배포하기 전에 이 목록을 하나씩 확인하세요.
Claude Code에게 "이 체크리스트 기준으로 전체 점검해줘"라고 요청하면 됩니다.

---

## 1. 입력값 검증

- [ ] 사용자 입력을 받는 모든 곳(폼, 입력창, URL 파라미터)에 검증 로직이 있는가?
- [ ] 프론트엔드(JS)와 백엔드(Flask) 양쪽에서 검증하는가?
- [ ] 입력 길이 제한이 있는가? (최소/최대)
- [ ] 빈 값, 공백만 있는 입력을 막는가?
- [ ] 이메일, URL 등은 형식 검증을 하는가?

## 2. SQL 인젝션 방어

- [ ] 모든 SQL 쿼리가 파라미터화 방식(?)을 사용하는가?
- [ ] f-string으로 쿼리를 만드는 곳이 없는가?

## 3. XSS 방어

- [ ] 템플릿에서 `| safe` 필터를 사용하는 곳이 없는가?
- [ ] 사용자 입력을 화면에 표시할 때 이스케이프 처리가 되는가?
- [ ] JavaScript에서 innerHTML 대신 textContent를 사용하는가?

## 4. 비밀 정보 관리

- [ ] API 키, Secret Key가 코드에 하드코딩되어 있지 않은가?
- [ ] 모든 비밀 정보가 os.getenv()로 처리되는가?
- [ ] .env 파일이 .gitignore에 포함되어 있는가?
- [ ] Railway Variables에 필요한 환경변수가 전부 설정되어 있는가?

## 5. 비밀번호

- [ ] 비밀번호를 해싱(generate_password_hash)해서 저장하는가?
- [ ] 평문으로 저장하는 곳이 없는가?
- [ ] 비밀번호 최소 길이 제한이 있는가?

## 6. 에러 처리

- [ ] API 호출, DB 접근, 외부 요청에 try/except가 있는가?
- [ ] 사용자에게는 친절한 메시지, 서버 로그에는 상세 에러를 기록하는가?
- [ ] 에러 발생 시 서버 내부 정보(line 번호, 코드 등)가 사용자에게 노출되지 않는가?
- [ ] 보안 관련 검증 실패 시 "일단 차단"을 하는가? (통과시키지 않는가?)

## 7. 배포 환경

- [ ] debug=False로 설정되어 있는가?
- [ ] HTTPS가 적용되어 있는가? (Railway는 자동 적용)
- [ ] FLASK_SECRET_KEY가 고정값으로 설정되어 있는가? (os.urandom 폴백에 의존하지 않는가?)

## 8. 코드 리뷰

- [ ] Claude Code에게 보안 관점 코드 리뷰를 요청했는가?
- [ ] 심각도 '중간' 이상의 이슈를 전부 수정했는가?
