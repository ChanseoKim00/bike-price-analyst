# Bike Price Analyst — 로드맵

## 서비스 개요

자전거 판매 페이지 URL을 입력하면 부품 개별 구매가 대비 얼마를 절약하는지 알려주는 분석 툴.
트래픽을 모아 광고 수익과 구독 수익을 병행하는 SaaS 모델로 운영한다.

---

## 플랜 구조

| 플랜 | 가격 | 제공 기능 |
|------|------|-----------|
| **Continental** | 무료 | 총 절약 금액만 표시 / 광고 있음 |
| **Pro** | 유료 (기본) | Continental 모든 정보 + 개별 부품 가격 표시 / 광고 제거 |
| **World Tour** | 유료 (풀옵션) | Pro 모든 정보 + 개별 부품 가격 변화 추이 그래프 + 가격 알림 |

---

## 수익 모델

- **초기**: 무료 트래픽 확보 + 광고 수익으로 API 비용 충당
- **중기**: 구독제(Pro/World Tour) 도입으로 안정적 수익 확보
- **장기**: 자전거 수입사/공식 대리점 B2B 협업 (할인 이벤트, 공식 가격 등록)

## 개발 로드맵

### Phase 1 — JS 렌더링 사이트 지원
> 네이버 스마트스토어 등 JavaScript로 렌더링되는 사이트 지원

- Playwright 도입 (Railway 환경 호환성 검토)
- 기존 requests 방식과 병행 운영 (JS 렌더링 필요 여부 자동 감지)
- 지원 사이트 범위 대폭 확대

### Phase 2 — AI 정보 추출 정확도 개선
> 구동계, 모델명 등 핵심 정보 추출 실패율 감소

- ai_analyzer.py 프롬프트 지속 강화
- 추출 실패 케이스 로깅 및 분석
- part_name_normalized 일관성 모니터링

### Phase 3 — 플랜 구조 구현
> Continental / Pro / World Tour 플랜 분리

- users 테이블에 plan 컬럼 추가 (continental/pro/world_tour)
- 플랜별 기능 접근 제어 구현
- 플랜 소개 페이지(/pricing) 제작

### Phase 4 — Stripe 결제 연동
> Pro / World Tour 유료 플랜 결제 기능

- Stripe Checkout 연동
- Webhook 처리 (결제 완료 → plan 업데이트)
- 구독 만료 처리 (plan_expires_at 기반)

### Phase 5 — 가격 자동 갱신 워커 + 가격 이력 저장
> TTL 만료 부품 자동 재조회 + 가격 변화 추이 데이터 축적

- 백그라운드 워커 구현 (Celery + Redis 또는 Railway Cron)
- part_price_history 테이블 추가 (가격 변화 이력 저장)
- World Tour 플랜용 가격 변화 추이 그래프 (Chart.js)

### Phase 6 — 가격 알림
> 설정 가격 이하로 내려오면 이메일/앱 알림

- 알림 설정 UI (자전거별 목표 가격 입력)
- 가격 체크 워커 (Phase 5 워커와 연동)
- 이메일 발송 (SendGrid 또는 SMTP)

## 현재 알려진 한계점

| 한계 | 원인 | 해결 계획 |
|------|------|-----------|
| JS 렌더링 사이트 미지원 | requests는 정적 HTML만 가져옴 | Phase 1에서 Playwright 도입 |
| AI 정보 추출 실패 | 일부 사이트 구조에서 핵심 정보 누락 | Phase 2에서 지속 개선 |
| 가격 정확도 | AI 웹 검색 기반이라 편차 존재 | Phase 5 워커로 정기 갱신 |
| part_name_normalized 중복 | AI 출력 불일치 | 프롬프트 강화 + 전방 일치 검색으로 완화 중 |
| 프레임셋 가격 조회 불가 | 공식 대리점 한정 판매 | DB 수동 입력으로 운영 |
| 분석 중 백엔드 취소 불가 | gunicorn 동기 워커 한계 | Phase 5 Celery 도입 후 해결 |
